Options

Option values can be passed into ngCss in a few places: 
  1. <div style="color: color;" ng-css="{ prelink: 'myCss', live: false }">...</div>

    overrides: 
  2. <style type="text/css" id="myCss" ng-css='{ live: true }' />...</style>

    or 
    <link rel="stylesheet" type="text/css" id="myCss" href="css/site.css" ng-css='{ live: true }' />

    overrides:
  3. The default values described below (which can be overridden).
In the example above, 
options.live
will be set to 
false
within the context of the 
DIV
as the 
DIV
tag's option values override the 
STYLE
/ 
LINK
tags' option values. 
options.live
will be set to 
true
within the context of the 
STYLE
/ 
LINK
tags' which overrides the default value (
false
).

prelink

VARIANT (see valid values below) false Callback hook before the directive wires up 
$scope
linking. This hook allows you to modify the 
$scope
that will be in effect for the directive and/or the 
$element
prior to rendering the directive: 
  • falsey
    - isolate scope. The directive will operate with its own isolated 
    $scope
    . 
  • string
    (as a HTML Element.id). The 
    string
    will be interpreted as an ID and the 
    $scope
    in effect for the referenced element will be imported. 
  • object
    (as an angular.element (jqLite/jQuery) or DOM element). The 
    $scope
    in effect for the referenced element will be imported. 
  • object
    (as a Javascript Object). The passed 
    object
    will be 
    extend
    ed into the directive's isolated 
    $scope
    . 
  • function
    (as a function object). This 
    function
    will be called with 4 arguments: 
    1. fnCallback
      (as a function object). The function to call on completion (e.g. 
      fnCallback();
      or 
      fnCallback($newScope);
      ) 
    2. $scope
      (as an Angular Scope). The default isolate scope in effect for the directive. 
    3. $element
      (as an angular.element (jqLite/jQuery)). The directive's 
      $element
      . 
    4. $link
      (as an angular.element (jqLite/jQuery) or 
      undefined
      for non-LINK tags). The replaced 
      LINK
      element.
    If you do not call 
    fnCallback
    the directive will not be properly wired up.
    If you wish to replace the default isolated 
    $scope
    with a 
    $newScope
    , pass it in as the first argument in the 
    fnCallback
    function (e.g. 
    fnCallback($newScope);
    ). The 
    $newScope
    object MUST be an Angular Scope, else the directive will not function.
    If a 
    $link
    is passed you can safely 
    $compile
    the 
    $element
    , though it is advisable to remove the 
    ng-css
    attribute before compilation (e.g. 
    $element.attr("ng-css", "")
    ) to avoid an unnecessary second call to ngCss.
Each 
ng-css
attributed 
LINK
tag is replaced with a 
STYLE
tag with only the 
ng-css
attribute copied across. If you require additional attributes or processing to apply to these created 
STYLE
tags, you can make any necessary modifications to the 
$element
within a 
prelink
function
.

script

VARIANT (see valid values below) false The Javascript scope that the ngCss-embedded Javascript will be 
eval
uated within: 
  • falsey
    - embedded Javascript is not evaluated. 
    SCRIPT
    tags embedded within the CSS are ignored. 
  • truthy
    - iFrame sandboxed Javascript. 
    SCRIPT
    tags embedded within the CSS will be evaluated within a sandboxed window object exposing only the 
    $scope
    .* Evaluated Javascript is cached within the sandbox and is not re-evaulated during reprocessing. 
  • "local"
    - non-"use strict" closure scope. Javascript is evaluated within the host window object with full access to global resources, but once execution is complete embedded Javascript resources are no longer accessible unless set (or leaked) into the global scope.

live

BOOLEAN false Specifies if changes made within the 
$scope
are to be automatically reflected within the CSS. When set to 
false
, the custom 
$broadcast
event updateCss is configured (e.g. 
$rootScope.$broadcast('updateCss');
) and can be triggered via the factory's 
ngCss.updateCss()
or via 
$rootScope.$broadcast('updateCss');
.

async

BOOLEAN true Specifies if 
LINK
tags are to be processed asynchronously. If ordering matters in the execution of the embedded 
SCRIPT
s, then this should be set to 
false
.
* The 
IFRAME
's 
window.parent
is set to 
null
. 

style
Attributes

Variable
s can be used within any tag's 
style
attribute (but 
SCRIPT
tags are not permitted). When applying ngCss to a 
style
attribute, you have two options when setting the value of 
ng-css
's in-line options: 
  • <div style="color: color;" ng-css="myCss">...</div>

    This will import the 
    $scope
    and option values from the HTML element with the ID myCss (e.g. "myCss" overrides 
    options.prelink
    ). This element should be a 
    ng-css
    attributed 
    LINK
    or 
    STYLE
    tag, but any Angular element's 
    $scope
    can be imported. 
  • <div style="color: color;" ng-css="{ prelink: 'myCss', live: false }">...</div>

    This will also import the 
    $scope
    /option values from ID myCss while overriding the value of 
    options.live
    to 
    false
    . This is useful when you want to use a 
    Variable
    within your 
    style
    attribute but do not require it to be live-bound like its source (as used in 2 of the 3 color profile selection boxes above to demonstrate the concepts).
Only the 
style
attribute is effected by ngCss in this context using a non-isolate 
$scope
(even when the referenced element is!). This is accomplished by a bit of pre- and post-
$compile
slight-of-hand between the directive's 
template
and 
link
functions and custom linking code via a 
$watch
or 
$on
.

How To Override The Default Option Values

If you wish to override any of the default option values you can call 
ngCss.defaults
, passing in the new default option values like so: 
var ngCn = angular.module("ngCn", ["ngCss"]);

ngCn.controller("ExampleController", ["$scope", "ngCss", function ($scope, ngCss) {
    ngCss.defaults({ script: true });
}]);
In the example above, we reset the default value for 
options.script
within ngCss to 
true
.

Getting Started

Assuming the default option values are in effect as described above:
  1. Download ngCss From GitHub
  2. Include the ngCss 
    SCRIPT
    tag in the 
    HEAD
    of your HTML document below your Angular 
    SCRIPT
    tag: 
    <script type="text/javascript" src="angular.min.js"></script>
<script type="text/javascript" src="angular.ngcss.min.js"></script>
  3. Include the attribute 
    ng-app="myAngularApp"
    in your HTML element (e.g. 
    <html ng-app="myAngularApp">
    ) and setup 
    myAngularApp
    to include the ngCss factory like so: 
    <script>
    var myAngularApp = angular.module("myAngularApp", ["ngCss"]);
</script>
  4. Add the 
    ng-css
    attribute to the LINK and/or STYLE elements we want to process: 
    <style id="myCss" type="text/css" ng-css>...</style>

<link id="myExCss" rel="stylesheet" type="text/css" href="css/site.css" ng-css/>
  5. Define the 
    options
     used by ngCss (if any): 
    <style id="myCss" type="text/css" ng-css="{ script: true, live: true }">...</style>

<link id="myExCss" rel="stylesheet" type="text/css" href="css/site.css"
    ng-css="{ script: true, live: true }"
/>
  6. Edit the CSS within the LINK and/or STYLE elements to use ngCss like this: 
    /*
<script>
    $scope.color = 'blue';
</script>
*/
DIV {
    color: /*color*/;
}
  7. (Optional) Consume ngCss (or any other 
    $scope
    's) 
    Variable
    s within any element's 
    style
    attribute like this: 
        <span style="color: color;" ng-css="myCss">...</span>

    <a href="http://opensourcetaekwondo.com/ngcss" style="color: color;"
        ng-css="{ prelink: 'myExCss', live: false }"
    >...</span>
    Technically, you can import any Angular element's 
    $scope
    into the 
    style
    attribute, but the intention is for you to reference 
    ng-css
    attributed 
    LINK
    or 
    STYLE
    elements.
NOTE: You are also permitted to use the attribute name 
data-ng-css
in place of 
ng-css
if you want to be HTML5 compliant.

Gotchas

  • It is recommended to comment the 
    Variable
    s (e.g. 
    /*Variable*/
    ) as the braces used by Angular are also used within CSS and these additional non-CSS-valid braces can cause issues with CSS editors. But... this will require you to modify your ngCss-enabled *.css files before CSS minimization as the commented 
    Variable
    s will likely be removed during minimization.
  • Any instances of 
    {{
    followed by 
    }}
    within your CSS will be eaten by 
    $interpolate
    or will otherwise cause errors. Make sure you have whitespace between all braces that are not Variables.
  • As 
    LINK
    tag CSS files are read via AJAX, the limitations of 
    XMLHttpRequest
     can restrict where you serve your CSS files from. If your CSS files exist on the same server, this is a non-issue.
  • Each 
    LINK
    tag is replaced by a new 
    STYLE
    tag that contains the modified CSS file contents. This process can break some relative URLs within your CSS files (e.g. 
    background-image: url("../img/a.png");
    ). It is therefore recommended that you use absolute URLs within your CSS files (e.g. 
    background-image: url("/img/a.png");
    ) in order to avoid any such errors.
  • Embedded 
    SCRIPT
    s are all evaluated in series at initialization prior to CSS modification, so the following will not work as expected: 
    /*<script>
    $scope.color = 'blue';
</script>*/
DIV {
    color: /*color*/; //# expecting 'blue' but gets 'green'
}

/*<script>
    $scope.color = 'green';
</script>*/
SPAN {
    color: /*color*/; //# expecting 'green'
}
    At initialization 
    $scope.color
    will be set to "blue" then "green" prior to any CSS modification, so the 
    color
    of both DIV and SPAN will be set to "green".
  • IE8+ is supported by ngCss out-of-the-box.
    The required "advanced" JavaScript functionality is:
    • XMLHttpRequest
    • getElementsByTagName and createElement

Side Effects

  • Each 
    LINK
    tag is replaced by a new 
    STYLE
    tag that contains the modified CSS file contents. This is done so that the original unprocessed CSS referenced by the 
    LINK
    tag does not apply to the webpage.
  • Any ngCss attributed tags without a defined HTML Element.id has a unique ID (formatted as ngCss###) added so ngCss can track it internally.
  • If you have elected to have a 
    script
    of 
    local
    , ngCss-embedded Javascript has the ability to modify the global scope.
  • And of course... the CSS within ngCss attributed tags is modified based on the embedded Variables and related 
    $scope
    .
